Real-Time Healthcare Monitoring System

Application 6 Project by Sara Lucia Vargas-Cortes

Project Overview

This project is a real-time healthcare monitoring system implemented using FreeRTOS on an ESP32. The system reads a simulated heart sensor using a potentiometer and detects when the signal exceeds a critical threshold.

Real-Time Task Structure

Sensor Task

Runs every 100 ms to detect abnormal sensor values.

Hard Real-Time Priority 2
Event Handler Task

Receives queue data and responds to alert events.

Hard Real-Time Priority 2
Button Task

Uses an ISR and binary semaphore to acknowledge the alarm.

Hard Real-Time Priority 3
Heartbeat Task

Blinks the green LED to show that the system is alive.

Soft Real-Time Priority 1

Key Features

Deadline Guarantee

Critical tasks run at higher priority so they are not delayed by lower-priority tasks.

Mutex Protection

A mutex protects serial printing and prevents multiple tasks from printing at the same time.

Counting Semaphore

A counting semaphore stores sensor alert events when the signal exceeds the threshold.

Binary Semaphore + ISR

The button interrupt gives a binary semaphore so the acknowledgement task can respond quickly.

Queue Communication

A queue safely transfers sensor readings from the sensor task to the event handler task.

Persistent Alarm State

The red LED alarm continues blinking until medical staff acknowledge it with the button.
