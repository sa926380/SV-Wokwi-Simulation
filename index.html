Real-Time Healthcare Monitoring System

Application 6 Project by Sara Lucia Vargas-Cortes

Project Overview

This project is a real-time healthcare monitoring system implemented using FreeRTOS on an ESP32. The system reads a simulated heart sensor using a potentiometer and detects when the signal exceeds a critical threshold.

Real-Time Task Structure

Sensor Task

Runs every 100 ms to detect abnormal sensor values.

Hard Real-Time High Priority
Event Handler Task

Responds to alerts and controls alarm behavior.

Hard Real-Time High Priority
Button Task

Uses an ISR to immediately detect user acknowledgement.

Hard Real-Time
Heartbeat Task

Blinks a green LED to show that the system is alive.

Soft Real-Time Low Priority

Key Features

Deadline Guarantee

Critical tasks run at higher priority so they are not delayed by lower-priority tasks.

Mutex Protection

A mutex protects serial printing and prevents race conditions.

Counting Semaphore

A counting semaphore tracks sensor events when the signal exceeds the threshold.

Binary Semaphore + ISR

Button input is handled immediately using an interrupt and binary semaphore.

Queue Communication

A queue safely transfers sensor data between tasks.

Timing Verification

Timestamps and the logic analyzer show the response time is under a few milliseconds, well below the 150 ms hard deadline.

Persistent Alarm State

The red LED blinks continuously until the user acknowledges it with the button.
