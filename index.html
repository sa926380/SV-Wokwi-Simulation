Application 6 Real-Time Hardware Demo

A physical embedded system prototype demonstrating real-time task scheduling, hardware response, and timing verification.

Project Overview

This project demonstrates a real-time embedded system running on physical hardware. The system monitors inputs, responds to events, and uses task scheduling to produce predictable system behavior.

The goal of this project was to show how embedded software can coordinate hardware components while meeting timing requirements.

Demo Video

The video below shows the physical hardware setup, system behavior, LED outputs, and timing verification.

Key Features

Physical Hardware

Implemented and tested using a real embedded hardware setup.

Real-Time Scheduling

Uses scheduled tasks to manage timing and system behavior.

Event Response

Responds to hardware input through LEDs and system actions.

Timing Verification

Uses serial output/logs to verify response timing.

What I Learned

This project helped me better understand how real-time systems use tasks, timing, and hardware interaction to create predictable behavior. I also learned how important it is to test the system on physical hardware because real components can behave differently than simulations.
